Prework Study Guide

HTML

  • The "head" element contains information about the webpage.
  • The "body" element represents the visible content shown to the user.
  • "style" defines the CSS styles associated with a website
  • "base" defines the URL for a webpage
  • "meta" defines metadata such as character set, description, keywords, author. types of information that can be stored in the meta tag
  • The "charset" element sets our character encoding.

CSS

  • Cascading Style Sheets, is the language used to describe the presentation of a website that’s created with HTML and how each element on a webpage is rendered.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • GITcommands always start with git and are used to save and track versions of your code
  • GIT CHEAT SHEET : https://education.github.com/git-cheat-sheet-education.pdf
  • Repository (repo) A folder for your project that contains all the files
  • Branch workspace were you can make changes that don’t affect the live site or work branch repository
  • Markdown (.md) A way to write for the web using plain text; easily converts to HTML
  • Commit changes Whenever you add, edit, or delete a file, you’re making a commit. Adds changes to your branch
  • Pull Request (PR) Asking for changes in your branch to merge into another branch. Way to discuss and review your work with others
  • Merge when a PR is approved is “pulled in”
  • Issue Report a new task or content fixes
  • This Markdown Cheat Sheet https://www.markdownguide.org/cheat-sheet/  - is a tool to help users transcribe plain text language into GitHub code.

JavaScript

  • JavaScript is a lightweight programming language that is used to provide logic and interactivity for websites.
